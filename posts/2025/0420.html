Node.js完整部署指南 ​

版本选择建议

推荐生产环境使用LTS版本（长期支持版），学习开发可使用Current版体验最新特性

一、环境准备 ​

1. 系统要求 ​

  • Windows：需管理员权限（建议Windows 10/11）
  • macOS：10.13及以上版本
  • Linux：Ubuntu/Debian系推荐使用apt管理

重要提示

安装前请确保系统已安装最新安全补丁，避免出现依赖冲突

二、安装流程 ​

1. 基础安装 ​

bash
Linux用户安装命令（Debian系）
sudo apt-get update && sudo apt-get install nodejs npm
1
2
点击查看Windows特殊配置

若遇到MSBuild错误，需安装：

  1. Visual Studio Build Tools
  2. Python 2.7（新版本可能不兼容）
  3. 勾选"C++桌面开发"组件

2. 验证安装 ​

bash
node -v  
npm -v
1
2

成功验证

当终端显示版本号（如v18.16.0），说明基础环境已就绪 若提示命令未找到，请检查环境变量配置

三、深度配置 ​

1. 路径优化 ​

bash
npm config set prefix "X:\nodejs\node_global"
1

路径安全警告

  1. 不要使用中文路径（如"文档/nodejs"）
  2. 避免特殊字符（建议纯英文路径）
  3. 不要直接设置在系统盘根目录

2. 镜像加速 ​

bash
npm config set registry https://registry.npmmirror.com/
1

镜像切换技巧

  • 测试阶段可使用--registry临时参数
  • 企业私有仓库建议使用npm login认证

四、常见问题 ​

1. 权限错误解决方案 ​

权限处理流程

  1. 右键安装目录 → 属性 → 安全
  2. 编辑 → 选择Users组
  3. 勾选"完全控制" → 应用

2. 模块加载失败 ​

bash
npm rebuild
1

高危操作警告

以下命令需谨慎使用：

bash
sudo rm -rf node_modules/  # 强制清除依赖
npm cache clean --force   # 彻底清空缓存
1
2

五、进阶配置 ​

1. 多版本管理（nvm） ​

多版本切换建议

  • 项目目录创建.nvmrc文件指定版本
  • 使用nvm install --lts自动安装最新LTS版
  • 通过nvm alias default 18.16.0设置默认版本

2. 核心工具安装 ​

bash
npm install -g yarn typescript
1
点击查看完整工具列表

推荐开发工具链：

  • nodemon：文件监控重启
  • pm2：生产环境进程管理
  • eslint：代码质量检查
  • webpack：项目打包工具
